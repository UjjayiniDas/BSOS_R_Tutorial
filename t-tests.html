Hypothesis Tests

One of the basic inferential analyses you can do with data is a hypothesis test. R is very helpful for performing t-tests, with built-in functions allowing you run a variety of different types of tests, including one sample and two sample tests, with ease.

We will be using the teengamb dataset from package faraway in this tutorial to look at how to use R to do some basic t-tests.The teengamb data frame has 47 rows and 5 columns. A survey was conducted to study teenage gambling in Britain. For documentation of the dataset, refer to the link.

Let‚Äôs have a look at the dataset quickly before moving on.

As a reminder, the steps to a hypothesis are:

  1. State your hypotheses.

  2. Set the criterion.

  3. Collect sample data.

  4. Calculate the test statistic.

  5. Evaluate and interpret.

Most of these steps will be done outside of R, and in fact, they can generally only be done outside of R. However, R can be very useful for the computational part, step 4. We won‚Äôt go over every single step here and focus mostly on the R part, so remember to keep in mind that the code is only one small portion of the hypothesis testing process.

Hypothesis Test Setup

Suppose an official from the Government is trying to prepare a report on how much teenagers use on gambling per year in order to develop some public policy measure. He writes in the report that the average expenditure for teenage gambling is 7 pounds a year. Assuming that the 47 teenagers in the teeengamb dataset are an independent and random sample, how might the official test whether his claim was correct(that is, the mean is 10)?

The hypotheses would look like this.

\[H_0: \mu = 7 \] \[H_1: \mu \ne 7\] In the examples in this tutorial, we will use an alpha level of 0.05. In general, the R code won‚Äôt change depending on the alpha level that you decide, though the code will change if you want different confidence levels for confidence intervals.

Using t.test()

We can do the t-test by hand by calculating the test statistic and finding the critical value, but R also has a convenient way of doing it using just the data with the t.test function. You do need to make sure you still go through all of the steps of performing a hypothesis test though ‚Äì the t.test() function will perform the test for you, but you need to make sure to provide all of the appropriate arguments.

Let‚Äôs look at doing a simple two-sided test, with a null hypothesis that \(\mu = 100\). We provide the vector of data points, as well as an argument providing the null value. In this case, since the value is in pounds, we use mu = 7. Always keep in mind the unit in which the variable is reported while setting the value for the null hypothesis.

t.test(teengamb$gamble, mu = 7)
## 
##  One Sample t-test
## 
## data:  teengamb$gamble
## t = 2.6759, df = 46, p-value = 0.01029
## alternative hypothesis: true mean is not equal to 7
## 95 percent confidence interval:
##  10.04766 28.55447
## sample estimates:
## mean of x 
##  19.30106

The output provides the test statistic, degrees of freedom, and the p-value in the top line. In this case, our p-value was 0.01029, so we would reject the null hypothesis (remember, our alpha level was 0.05). Below, we get a few more pieces of information, such as the actual sample mean.

By default, this does a two-sided test. What if we wanted to do a one-sided test? We can use the alternative argument to change it.

# These are how to do one-sided tests.
# Note: This is just to show that it's possible! Don't 
# change the sidedness after the fact like this!
t.test(teengamb$gamble, mu = 7, alternative = "greater")
## 
##  One Sample t-test
## 
## data:  teengamb$gamble
## t = 2.6759, df = 46, p-value = 0.005146
## alternative hypothesis: true mean is greater than 7
## 95 percent confidence interval:
##  11.58416      Inf
## sample estimates:
## mean of x 
##  19.30106
t.test(teengamb$gamble, mu = 7, alternative = "less")
## 
##  One Sample t-test
## 
## data:  teengamb$gamble
## t = 2.6759, df = 46, p-value = 0.9949
## alternative hypothesis: true mean is less than 7
## 95 percent confidence interval:
##      -Inf 27.01797
## sample estimates:
## mean of x 
##  19.30106

Note that this doesn‚Äôt provide the conclusion of the test for you. You need to make the correct conclusion from these results. Basically, you just need to take the p-value and compare it to the alpha level like before.

Lastly, notice that the output gives you information about the confidence interval. You can use the t.test() function to create confidence intervals as well. Just make sure you use the conf.level argument to specify the type of confidence interval you want.

t.test(teengamb$gamble, conf.level = 0.99)
## 
##  One Sample t-test
## 
## data:  teengamb$gamble
## t = 4.1986, df = 46, p-value = 0.0001217
## alternative hypothesis: true mean is not equal to 0
## 99 percent confidence interval:
##   6.948697 31.653430
## sample estimates:
## mean of x 
##  19.30106

If you‚Äôre just making a confidence interval, you can ignore the rest of the information. Here, I ran t.test() without anything for mu ‚Äì this gave me a test comparing it to 0, but I don‚Äôt need that information since all I need is the confidence interval.

Two Sample t-tests

Independent Samples t-test

Doing an independent samples t-test can be done one of two ways. If you have the data from the two samples in two vectors, you can simply use those vectors to run the t-test. In a data frame, this would mean that the two samples are stored as two separate columns.

If we wanted to run a t-test to see if there was as significant difference between socioeconomic status score i.e.¬†status and verbal score i.e.¬†verbal, then we could use the following code.

t.test(teengamb$status,teengamb$verbal)
## 
##  Welch Two Sample t-test
## 
## data:  teengamb$status and teengamb$verbal
## t = 15.231, df = 47.064, p-value < 2.2e-16
## alternative hypothesis: true difference in means is not equal to 0
## 95 percent confidence interval:
##  33.47976 43.66918
## sample estimates:
## mean of x mean of y 
## 45.234043  6.659574

What if the data were stored in a way such that all the values were in one column, with the corresponding groups in another column? That is, for example, what if we wanted to do an independent samples t-test for gambling expenditure of teenagers based on their gender?

We can use formula notation in this case.

t.test(gamble ~ sex, data = teengamb)
## 
##  Welch Two Sample t-test
## 
## data:  gamble by sex
## t = 3.6227, df = 28.503, p-value = 0.001123
## alternative hypothesis: true difference in means between group 0 and group 1 is not equal to 0
## 95 percent confidence interval:
##  11.27085 40.54758
## sample estimates:
## mean in group 0 mean in group 1 
##       29.775000        3.865789

On the left of the tilde (~), we have the numerical variable that we want to use the mean of. On the right side, we put the categorical variable that denotes which person is in which group (i.e.¬†male or female). The data = teengamb argument, as before, denotes which data to pull these columns from.

Paired t-test

Above, we used an independent sample t-test to look at the difference between socioeconomic and verbal scores. This isn‚Äôt quite how we should approach this, though. Remember, each teenager was assigned both the scores, so this is actually a paired t-test, since the scores on each variable (socioeconomic status, verbal ability) are paired according to which teenagers got them. We can do a paired t-test by adding an argument paired = TRUE to the t.test function.

t.test(teengamb$status, teengamb$verbal, paired = TRUE)
## 
##  Paired t-test
## 
## data:  teengamb$status and teengamb$verbal
## t = 16.173, df = 46, p-value < 2.2e-16
## alternative hypothesis: true mean difference is not equal to 0
## 95 percent confidence interval:
##  33.77341 43.37553
## sample estimates:
## mean difference 
##        38.57447
